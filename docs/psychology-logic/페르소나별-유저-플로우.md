## 공통 진입 흐름 (모든 페르소나 동일)

```
앱 실행
 └─ 익명 로그인 (useAuth)
    └─ /onboarding/consent          (동의)
       └─ /onboarding/duration      (이별 기간)
          └─ /onboarding/persona Q1~Q6 (+외부복잡도)
             └─ 점수 합산 → primary 페르소나 확정 (usePersonaStore)
                └─ /onboarding/persona/intro (페르소나 인트로 카드 1회)
                   └─ (tabs)/index = 홈
```

| 단계 | 파일 | 페르소나 영향 |
|------|------|---------------|
| 진입 라우팅 | `app/index.tsx:19-46` | 동일 |
| 동의·기간 | `app/onboarding/consent.tsx`, `duration.tsx` | 동일 |
| 분류 질문 | `app/onboarding/persona/index.tsx` | Q1~Q6 응답 → 점수 |
| 분류기 | `utils/personaClassifier.ts` | 합산 후 최고점 = primary |
| 인트로 카드 | `app/onboarding/persona/intro.tsx` | **페르소나마다 다름** ↓ |
| 홈 진입 | `app/(tabs)/index.tsx` | PersonaPriorityCard·ContactUrgeChip 등 페르소나 분기 |

> ⚠️ **라벨 비노출 원칙**: 사용자 화면에는 페르소나 코드(P01~P20)나 진단명-유사 어휘(불안형·회피형·트라우마 본딩 등)를 직접 노출하지 않음. 본 문서는 *내부 개발자 참고용*.

## P01 — 자기 판단 손상형

| 항목 | 값 |
|------|-----|
| 유형 | 🟡 C 감정 조절 |
| 1차 장애물 | 현실 검증 기준점 회복 |

### 분류 신호
- Q2 "내가 너무 예민한가?" 단독 → +2
- Q2 ① + Q6 "방향이 자주 바뀜" 동시 → **+5** (대체 규칙)
- Q6 단독 → +1

### 인트로 카드 (`shield`)
> "먼저 한 가지 알려줄게"
> 관계에서 자기 판단이 흔들리는 패턴은 *통제·반복적 현실 부정*에서 비롯되곤 해. 네가 너무 예민한 게 아니야.
> 앞으로 며칠은 *해석 말고 사실 정리*에 집중해보자.

### 홈 우선 카드 (`scale`)
- "오늘 하나, 사실 정리" / "해석 말고 그날 있었던 일만"
- 이동: `/about-me`

### 분기
| 트랙 | 동작 |
|------|------|
| 일기 질문 | `j_fact_only` **booster +20** (사실 기록 우선) |
| 일기 질문 차단 | `a_their_feeling`, `a_fix_possible` (상대 변호·관계 회복) |
| 분석 트랙 | **전체 차단** — `me.tsx` 분석 카드 비활성, deep link 가드 |
| 나침반 verdict 후미 | "너의 잘못이 아니야. 사실을 사실대로 본 거야." |
| 추억 미화 카테고리 | `happy`·`miss` 차단 (단절 의지 약화 방지) |
| about-me 정렬 | `reality_check` → `self_love` 우선 |

### GPT 톤
- **톤**: 사실 확인 우선, 단정 금지, 학대 패턴을 사실로 인정
- **금지 키워드**: "둘 다의 문제", "그 사람도 이유가", "소통해봐", "이해해주자", "오해였을"
- **프레이밍**: 제3자 관점·시간순 사건 묘사

### 한 줄 요약
> **해석을 빼고 사실만**. 분석 차단·미화 차단·"너의 잘못이 아니야" 강제로 자기 의심 루프를 끊는다.

> 학대 관계에서 떠나기 어려운 건 신경생리적 현상이야. 자책 대신 *접촉 단절일 카운터*로 객관화하자.

### 홈 우선 카드 (`award`)
- "단절 N일째" / "도파민 사이클을 끊는 카운터"
- 이동: `/about-me`

### 분기
| 트랙 | 동작 |
|------|------|
| 일기 질문 | `j_fact_only` **booster +18** |
| 일기 질문 차단 | `a_their_feeling`, `a_fix_possible` |
| 분석 트랙 | **전체 차단** (P01과 동일 — 미화 위험) |
| 나침반 verdict 후미 | "너의 잘못이 아니야. 사실을 사실대로 본 거야." |
| 추억 미화 카테고리 | `happy`·`miss` 차단 |

### GPT 톤
- **톤**: 생리적 현상 정상화, 비자책
- **금지 키워드**: "왜 못 떠나", "사랑 아니야", "한 번에 끊어", "의지가 약해"
- **프레이밍**: 도파민-코르티솔 사이클 / 단절 N일 카운트

### 한 줄 요약
> **의지 비난 금지, 신경생리로 설명**. 분석 차단·미화 차단·단절 카운터로 떠나기 어려움을 *객관화*.

## 다중 페르소나 처리 (`utils/personaResolver.ts`)

여러 페르소나가 동시에 점수 임계 통과 시 유형 기반으로 우선순위 결정:

| 주(primary) | 부(secondary) | 결과 |
|-------------|---------------|------|
| A 안전 (P14·P20) | * | **primary** (안전 우선) |
| * | A 안전 | **secondary** (부가 더 시급) |
| B 접촉 (P02·P08) | D 의미 | primary |
| C 조절 (P01·P03·P06·P10·P11·P19) | D 의미 | primary |
| C 조절 | B 접촉 | **merged** (둘 다 고려) |
| B 접촉 | C 조절 | merged |
| 그 외 | 그 외 | merged |

### Guard Overlay
부 페르소나의 *시간 게이트·환경 차단*은 effective와 별도로 덮어씌움. 예시:
- primary=P20, secondary=P03 → effective=P20 + **P03 가드**(D+21 추억 잠금, 0~5시 푸시 차단)

## 절대 규칙 교차 (CLAUDE.md)

페르소나 분기와 무관하게 항상 적용:

| 규칙 | 페르소나 영향 |
|------|---------------|
| 위기 신호 (3일 연속 mood 1~2 / 0~5시) → `EmotionalCheckModal` | 모든 페르소나 동일. P03의 새벽 푸시 차단과 별개(나가는 알림 차단 vs 들어오는 접근 감지) |
| C-SSRS 양성 → 결정 트랙 자동 잠금 | 모든 페르소나. P14/P20(A 안전 유형)은 가능성 더 높음 |
| 졸업 7일 유예 | 모든 페르소나 동일 |
| 페르소나 라벨 비노출 (`npm run lint:persona`) | 모든 사용자 화면 |
| 핫라인 정보 = `resources/crisis-hotlines.json` only | 페르소나별 *우선 핫라인* 매핑은 별도 |
