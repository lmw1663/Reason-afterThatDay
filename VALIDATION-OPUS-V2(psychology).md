**문제**: 기존 데이터가 많은 production 환경에서 `NOT NULL DEFAULT`로 ALTER TABLE은 row마다 update 발생 → 잠금 길어질 수 있음. Postgres 11+에서는 빠른 추가 가능하지만 명시 필요.

**보강 필요**: 
- 1단계: NULL 허용으로 컬럼 추가
- 2단계: 기존 row 백필 (배치)
- 3단계: NOT NULL 제약 추가

또는 명시적으로 "Postgres 11+ 환경에서 메타데이터만 변경" 주석.
