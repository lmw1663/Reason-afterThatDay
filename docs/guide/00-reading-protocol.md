# Reading Protocol (Token Saver)

개발 시 `CLAUDE.md` 전체를 매번 읽지 않고, 작업 목적에 맞는 문서만 선택해서 읽는다.

## 기본 원칙
- 먼저 `CLAUDE.md`의 "작업별 최소 참조"만 확인한다.
- 세부 규칙은 아래 가이드 중 필요한 파일만 추가로 읽는다.
- 구현 세부가 필요할 때만 `docs/reason_project_v2.md`를 펼친다.

## 권장 읽기 순서
1. 현재 작업 유형 결정 (UI/백엔드/AI/알림/로직/QA)
2. 해당 가이드 1~2개만 읽기
3. 구현 중 충돌이 생기면 관련 가이드만 추가로 읽기
