# 임상심리학자 관점 — Reason 앱 5차 평가

> **결론**: 잘 설계된 *결정 보조 + UX 안전장치* 앱이지만, **임상적 근거가 약한 가정** 위에 세워졌고 *치료적 관계의 모방*이라는 윤리적 위험이 누적되고 있다.
>
> **점수**: 임상 타당성 55% · 안전성 75% · UX 85%
>
> 이 평가는 V1~V4 검증(기획/사양/구현 동기화)과는 다른 축이다. V4가 *문서 정합성 94%*에 도달했더라도, **임상 타당성** 축에서는 별도의 결함이 누적되어 있다.

---

## 1. 가장 큰 구조적 문제 — 사용 중인 심리학 모델이 *낡았다*

### Kübler-Ross 5단계의 신뢰성 결함
`docs/psychology-tasks/00-overview.md`는 **Kübler-Ross 5단계 + Worden 4과제**를 통합 목표로 명시하지만, 임상심리학계에서 두 모델 모두 *비판받는 모델*이다.

| 모델 | 문제 |
|------|------|
| Kübler-Ross 5단계 | Maciejewski et al. (JAMA 2007) — 단계가 *순서대로* 진행되지 않음. 대부분 사람은 단계를 *건너뛰거나 비선형*으로 경험. 환자 사례에서 추출한 임상 관찰일 뿐 실증 모델 아님. |
| Worden 4과제 | *사별(bereavement)* 모델임. 이별은 임상적으로 "ambiguous loss" (Pauline Boss) — 상대가 살아 있고 SNS에서 보일 수 있다는 점에서 사별과 본질적으로 다름. |

### 더 적합한 근거 기반 모델 (사용 안 됨)
- **Bonanno의 회복탄력성(resilience) 궤적 모델** — 사별/이별 후 *대다수(50~60%)는 단계를 거치지 않고 회복*. 5단계 가정 자체가 일부 사용자에게 "내가 분노 단계를 통과 못해서 회복이 안 돼"라는 *잘못된 자기 진단*을 줌.
- **Sbarra & Hazan (2008) 이별 회복 모델** — 자기개념 명확성(self-concept clarity) 회복이 핵심 변수. 현재 앱은 *관계*를 분석하지 처음부터 *자기*를 분석하지 않음(6-11에서 뒤늦게 추가).
- **Slotter et al. (2010)** — "내가 누구인지 모르겠다"는 self-loss가 이별 후 우울의 핵심 매개변수. 앱의 "잡기/보내기" 차원은 self-loss를 측정하지 못함.

**처치**: docs/psychology-analysis.md와 task 문서들의 모델을 Bonanno + Sbarra 기반으로 재정렬. 5단계 메타포 의존도 낮추기.

## 2. *치료적 관계의 모방품* 윤리 문제 — 새 절대 규칙 후보

이 앱은 임상가가 아니지만 **임상가의 외형**을 가진다:
- GPT 공감 응답 (Rogerian 톤 모방)
- "회복 가망 67%" 같은 진단 수치
- "졸업" 메타포 (치료 종결의 모방)
- 매일 자기 관찰 일기 (CBT 자기모니터링 모방)

이건 *illusion of therapy* 문제다. 진짜 임상 도움이 필요한 사용자를 **전문 상담에서 멀어지게** 만들 수 있다.

### 현재 앱에서 누락된 임상 윤리 표준
| 표준 | 현재 |
|------|------|
| "이 앱은 전문 상담을 대체하지 않습니다" 영구 고지 | ❌ 없음 (위기 시 핫라인만 노출) |
| 데이터 보존/삭제 정책 명시 | ❌ 일기 = 매우 민감 데이터인데 OpenAI 전송 동의·보존기간 불명 |
| 임상 진단 도구 자제 | ❌ "재결합 가망 35%" 같은 *근거 없는 수치*를 보여줌 |
| 치료적 의존 경고 | ❌ 매일 들어와 GPT와 대화 = parasocial 의존 위험 |

**처치 (절대 규칙급)**:
- 모든 GPT 응답 화면 하단에 *작게라도* "전문 상담이 필요한 신호일 수 있어" 영구 노출 또는 옵트인 경로
- "회복 가망 %" 수치 표기 **삭제** 또는 *추세*로만 표시 (수치 자체가 의사과학)
- 일기 데이터의 OpenAI 전송 여부, 보존기간을 온보딩에서 *명시 동의*

## 3. "감정 온도 1~10"의 임상적 환원주의

V4까지 정교화된 `mood_score`(1~10)는 실제로 임상적으로 의미 있는 척도인가?

### 문제
- **PHQ-9, BDI-II** 같은 검증된 우울 측정 도구는 *증상 빈도*(잠/식욕/무가치감)를 묻지 *온도*를 묻지 않는다.
- 사용자는 "오늘 6점이었는데 7점이 됐어, 회복하고 있어"라고 *수치 추적에 집착*할 수 있음 — quantified self의 부작용.
- 6.4 → 6.7 같은 미세 변동이 *실제 임상 변화*를 의미하지 않음에도 차트는 *의미 있어 보이게* 그림.

### 처치
- 6-1에서 추가된 감정 라벨(분노/죄책/안도 등)을 *주축*으로 옮기고, mood_score는 *보조 지표*로 강등
- "mood_score = 6.4"가 아니라 *PHQ-2 (2문항 우울 스크리너)* 같은 검증된 짧은 척도를 D+14, D+30 등 *마일스톤*에만 노출
- 차트의 미세 변동을 부드럽게 처리 (이동평균 등) — 일희일비 방지

---

## 4. 7일 유예 = *임상*이 아닌 *UX* 안전장치임을 인정해야 한다

### 문제
이별 후 cognitive dissonance는 평균 **2~3개월** 지속(Sbarra 2011). 자기개념 회복은 **6개월~2년**. 7일은 *충동적 종결을 막는 UX 데드존*이지 *임상적 회복 기간*이 아니다.

그런데 앱은 7일 유예를 통과하면 "졸업"을 부여한다 — 이건 사용자에게 *7일 = 충분*이라는 잘못된 신호를 줌.

### 처치
- "졸업"이라는 단어 재고. 현대 grief work 이론은 *완결(closure)*이 아니라 *integration(통합)* 강조. 비탄은 끝나지 않고 *함께 살아가는 것*.
- 졸업 후 D+30, D+90 *재방문 게이트* — "졸업했다고 끝난 게 아니야. 가끔 흔들려도 정상이야" 메시지를 추후 자동으로 띄움
- 7일 유예의 정체를 사용자에게 *솔직하게* 알리기: "7일은 충동을 막기 위한 시간이지 회복 시간이 아니야"

## 5. 애착 유형/트라우마 이력 스크리닝 부재 — 가장 큰 *위험* 누락

현재 앱은 모든 사용자에게 *동일한 콘텐츠*를 제공한다. 그러나 임상적으로 다음 그룹은 완전히 다른 개입이 필요:

| 그룹 | 현재 앱 처치 | 필요한 처치 |
|------|------------|-----------|
| 불안 애착자 | 동일 일기 흐름 | "잡고 싶다"의 빈도 자체가 회복 지연 신호 — 횟수 노출 제한 필요 |
| 회피 애착자 | 동일 일기 흐름 | 감정 회피 패턴 → 신체 신호(physical_signals) 우선 입력 필요 |
| **가스라이팅/언어폭력 피해자** | "장점도 떠올려봐" 카드(6-8) | **장점 회상 = 가해 강화** — 이 사용자에게 *위험* |
| 자살 사고 이력자 | 7-4 위기 모달 | 이별과 *무관하게* 영구적 안전 계획 필요 |

### 처치 (P0급)
- 온보딩에 **2~3문항 트라우마 스크리너** 추가 (예: "관계에서 너의 의견이 무시당했다고 느낀 적이 있어?" / "헤어지자고 말했을 때 무서웠어?")
- 양성 응답 시:
  - 6-8 시점별 장점 회상 트랙 *비활성*
  - "지금 너에게 필요한 건 분석이 아니라 안전" 메시지 + 핫라인 연결 강화
  - GPT 프롬프트에 "이 사용자는 트라우마 이력 가능성 — 관계 미화 금지, 자기 보호 우선" 컨텍스트 주입

## 6. "잡고 싶어 / 보내고 싶어"는 *행동 의도*와 *애착 활성화*를 혼동한다

V3에서 추가된 6-2 수평축(원망↔애정)은 부분 해결책이지만 본질을 놓침.

### 임상적으로 정확한 차원
| 차원 | 측정 내용 | 현재 앱 |
|------|---------|--------|
| 행동 의도 (intent) | 연락하고 싶은가? | △ direction으로 일부 |
| 애착 활성화 (attachment activation) | 상대 생각이 *통제 불가능*하게 떠오르는가? | ❌ 6-5 떠오름 트랙이 부분 측정 |
| 정서 가치 (valence) | 좋게 vs 나쁘게 | ✅ 6-2 수평축 |
| 자기개념 명확성 | "내가 누구였는지 모르겠다" | ❌ 없음 |

행동 의도(잡고 싶다)와 애착 활성화(못 멈춤)는 다르다. 사용자는 *보내고 싶지만 머릿속에서 떠나지 않을* 수 있다. 현재 앱은 이걸 동일선상에 둠.

### 처치
- 6-5 떠오름 트랙의 *빈도/지속시간*을 측정해 "애착 활성화 지수"로 별도 표시
- 6-11에서 "헤어진 너 vs 사귀기 전 너 vs 지금 너"를 구별하는 자기개념 명확성 척도 (Slotter 11문항 단축형) 추가

## 7. 매일 일기 = *반추(rumination) 강화* 위험

Nolen-Hoeksema의 반응 양식 이론(response styles theory)은 명확하다: **반복적 자기 분석은 우울 위험을 높인다**.

현재 앱은 매일 일기를 *권장*하고 졸업 게이트(`canGraduate`)에 *5개 일기*를 요구. 이건 임상적으로 *반추를 강화*하는 게이트다.

### 처치
- "매일 쓰는 게 좋아"가 아니라 "**필요할 때만 써**"로 톤 전환
- 7일 연속 일기 작성 감지 시 **부드럽게 멈춤 권유**: "최근 매일 쓰고 있어. 오늘은 그냥 쉬어도 돼"
- 졸업 게이트의 *5개 일기*를 *5번의 자기 관찰*(어떤 형태든)로 완화

## 8. 확증편향 — 앱이 사용자 결정을 *고정*시킨다

현재 흐름:
- 사용자가 "잡고싶어"를 누름 → 다음 질문도 그 맥락
- `relationship_profile`은 누적 → 초기 입력이 후속 결과 *왜곡*
- AI 응답도 사용자의 현재 direction을 *공감* → 반박적 정보 0

이건 임상 면담의 정반대다. 면담자는 의도적으로 **반박적 가설(disconfirming hypothesis)**을 던져 사용자가 자기 입장을 *재검토*하게 만든다.

### 처치
- 주 1회 *방향 반대편* 질문 강제 노출: "오늘은 보내는 쪽 입장에서 한 줄 써볼래?"
- AI 응답에 *부드러운 재구성(reframe)* 1회 의무화: "그런데 한 발짝 떨어져서 보면…"
- relationship_profile 입력값에 *시점 가중치 감쇠* 적용 — 오래된 입력은 영향력 자동 감소

## 9. 데이터/프라이버시 — 임상 윤리 표준 미달

이별 일기는 **HIPAA 수준 민감 데이터**다. 앱은:
- ✅ RLS는 적용
- ❌ OpenAI 전송 동의 명시 부족
- ❌ 데이터 삭제 권리(GDPR 17조 / 개인정보보호법 36조) UI 부재
- ❌ "혹시 법정 분쟁이 생기면 이 데이터가 증거로 사용될 수 있다"는 고지 없음

### 처치
- 온보딩에 *명시적 동의*: GPT 전송, 보존기간(예: 졸업 +90일 자동 익명화), 삭제 권리
- 설정에 "전체 데이터 다운로드/삭제" 즉시 실행 가능 버튼

## 10. 종합 — 우선순위 재배열 (P-임상 트랙)

기존 P0~P3와 별개로, **임상심리학 관점에서 P-임상**이라는 트랙을 추가해야 한다.

| 임상 우선순위 | 항목 | 위험 수준 |
|---|------|---------|
| **임상-P0** | 트라우마/가스라이팅 스크리너 → 6-8 차단 | 사용자 가해 가능 |
| **임상-P0** | "전문 상담 대체 안 함" 영구 고지 + GPT 의존 경고 | 임상 윤리 위반 가능 |
| **임상-P1** | "회복 가망 67%" 수치 제거 또는 *추세*화 | 의사과학적 진단 |
| **임상-P1** | 매일 일기 권장 → "필요 시" 권장으로 톤 전환 + 7일 연속 작성 시 휴식 권유 | 반추 강화 |
| **임상-P1** | 데이터 동의/삭제 UI | 법적·윤리적 노출 |
| **임상-P2** | 자기개념 명확성(Slotter) 척도 추가 | 회복 핵심 변수 미측정 |
| **임상-P2** | 졸업 후 D+30, D+90 재방문 게이트 | "졸업=끝"의 잘못된 신호 |
| **임상-P3** | 5단계 모델 의존도 → Bonanno 회복탄력성 모델로 톤 재조정 | 잘못된 자기 진단 유도 |

## 11. 강점 — 인정해야 할 것

엄격하게 평가하지만, 다음은 임상적으로 **잘 설계된 부분**이다:

- ✅ "정답이 아니야" 영구 문구 — CBT 흑백사고 차단 표준
- ✅ 7-4 위기 모달 + 핫라인 분리 관리 — 임상 안전 기본
- ✅ 비난 금지 톤 — Rogerian 정합
- ✅ 7일 유예 + 알림 절제 — UX 안전장치로는 우수
- ✅ V4까지 진행된 *문서화 정합성* — 대부분의 정신건강 앱이 못 가지는 수준

## 결론

**"이 앱은 결정 보조 도구로는 좋고, 자해 방지 도구로는 적정하지만, *애도 회복 도구*로 본격 표방하기에는 임상 근거 보강이 더 필요하다."**

위 10개 항목 중 **임상-P0 두 개**(트라우마 스크리너, 치료 대체 고지)는 출시 전 반드시 처치되어야 한다. 나머지는 V5+ 로드맵.

V4까지의 작업은 **기획 → 사양 → 구현 가능성** 축으로는 94%지만, **임상 타당성** 축으로는 다시 채점하면 55% 정도. 두 축은 다른 문제이고, 둘 다 충족해야 진짜 회복 앱이 된다.
