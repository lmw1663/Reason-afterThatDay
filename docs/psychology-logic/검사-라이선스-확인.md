## 코드 활성화 토글

라이선스 미확인 검사는 다음 상수로 일괄 비활성:

```typescript
// constants/assessmentLicenses.ts (B-1·D-1에서 신설 예정)
// 키 명명은 supabase/migrations 022_assessments.sql의 instrument enum과 일치시킨다.
export const ASSESSMENT_AVAILABLE = {
  PHQ9:   false,  // ⬜ 표기 문구 확정 후 true
  PHQ2:   false,
  GAD7:   false,
  GAD2:   false,
  RSE:    false,  // ⬜ 인용 출처 표기 후 true
  CSSRS:  false,  // ⬜ Columbia 등록 후 true
  ECRR12: false,  // ⬜ 회신 후 true (DB enum: ECRR12)
  RRS10:  false,  // ⬜ 회신 후 true (DB enum: RRS10)
  PG13:   false,  // ⬜ 회신 후 true (TGI-SR+ 통합 검토 — enum 추가 시 별도 키)
} as const;
```

> **enum 동기화 주의**: TGI-SR+는 현재 D-1의 `instrument` enum 후보에 미포함. 라이선스 회신 후 사용 결정되면 마이그레이션 022에 추가하고 본 토글에도 별도 키 추가. PG-13과 통합한다면 별도 토글 불필요.

각 검사 활성화 시 본 표의 "회신 보존" 칸 채움 → 토글 true.

## 한국 임상 자문 + 운영 정책

라이선스 외 *한국어 번역본*의 임상 신뢰도 확인 필요:

- [ ] 임상심리 전문가 1~2명 섭외 (자문 계약)
- [ ] 한국어 번역본의 신뢰도·타당도 자료 수집
- [ ] PIPA 민감정보 처리 법무 자문

### 누락되기 쉬운 운영 항목

- [ ] **연령 게이트** — C-SSRS 등 위기 검사는 만 14세 미만 동의 능력 한계. 약관(A-1)의 만 14세 제한과 일관성 확인 + 미성년자 시도 시 분기
- [ ] **데이터 보존 vs 라이선스 충돌 검토** — 일부 라이선스는 응답 데이터 *외부 발표/활용* 제한. 5년 익명화 보존(`021_data_retention.sql`)이 라이선스 조건과 충돌하지 않는지 확인
- [ ] **라이선스 갱신 주기** — 유상 라이선스의 갱신 주기·연간 비용·자동 갱신 여부 추적표 작성 (예: 매년 4월 갱신)
- [ ] **라이선스 위반 시 즉시 OFF 절차** — 갱신 실패·기관 정책 변경 시 토글을 false로 되돌리고 사용자 데이터 처리 방안
