## 한 문장 평가

> **"이제 친구 같다. 다만 첫 만남이 여전히 길고, 가끔 약속을 안 지킨다."**

v1의 "안전한지 의심스럽다"는 인상은 사라짐. 반말 톤·매듭의 가역성·mini-mode 같은 escape hatch로 *옆에 앉아주는 친구* 느낌을 만들어냄. 다만 **온보딩 길이**, **late_night 약속 불이행**, **카드가 왜 떴는지 모름**의 세 가지가 사용자 신뢰를 미세하게 깎고 있음.

## 시나리오 1 — 첫 진입 (D+7, 23:30)

### 동선
```
앱 다운로드 → OAuth 로그인 → 동의(필수 3개) → 페르소나 Q1~Q6 +
C-SSRS 3항 → 관계 기간 → 현재 기분 → 홈 도착
```

### 좋았던 것
- 동의 화면 카피 *"네 이야기를 안전하게 다루기 위한 약관"* — 첫 인상부터 톤 명확.
- 페르소나 6문항이 *어세스먼트가 아닌 대화* 톤. C-SSRS 3항도 갑자기 무겁지 않게 들어가.
- 홈 도착 시 D+N 디스플레이가 헤딩으로 박혀있어 *"내가 어디 있는지"*가 즉시 잡힘.

### 거슬렸던 것
- **"동의하지 않을래" → "사용할 수 없어"** (`app/onboarding/consent.tsx:60~66`)
  - 이별 직후 첫 화면이 *차단* 어휘. PIPA상 불가피한 설계지만 "지금은 살펴보기만 해도 괜찮아" 톤으로 부드럽게 풀 수 있어.
- **온보딩 총 6단계**(동의·OAuth·페르소나·C-SSRS·기간·기분) — 무기력 사용자에게 첫 세션치곤 길어. 페르소나 6문항을 4문항으로 줄이거나 C-SSRS를 *D+1 푸시 이후* 분산하는 안 검토.

## 시나리오 2 — 일상 일기 작성 (D+12, 22:00)

### 동선
```
홈 → "오늘 일기 쓰기" → 1/4 감정온도 → 2/4 방향 →
3/4 스마트 질문 → 4/4 AI 응답 스트리밍 → 저장
```

### 좋았던 것
- mini-mode "감정 온도만 빠르게" 링크가 primary 아래에 작게 있어 *부담 없이 통과 가능*. (`app/(tabs)/index.tsx:317~328`)
- 1/4 화면의 *"여러 개 동시에 느껴도 돼"* — 양가감정 인정 카피. 작지만 큰 신호.
- 감정 라벨 선택 후 *"지금 그 감정, 정상적인 단계야"* 보라 강조 카드 — 정확히 필요한 순간의 normalizing.
- AI 응답 스트리밍 글자 효과 — *읽히는 느낌*. 일괄 출력 대비 살아있는 톤.

### 거슬렸던 것
- 신체 신호 collapsible은 좋은데, **펼쳤다가 다시 접는 토글이 없음**. (`app/journal/index.tsx:128~135`) 한 번 펼친 사용자가 *"역시 안 채울래"* 결정 시 다시 접을 길 없음. 의도된 선택이지만 *되돌릴 자유*는 신뢰의 기본.
- 4/4 AI 응답 후 저장 다음 *어디로 가야 하는지* 마무리감이 약하다. 홈으로 자동 복귀? 응답 화면에 머무름? 이 분기가 사용자 모델에 안 잡혀.

## 시나리오 4 — D+30, 매듭 권유 모달

### 카피 (탁월)
```
매듭 짓기 시작할까?

여기까지 와줘서 고마워.
그동안의 시간을 한 번 정리하는 매듭을 지을 수 있어.
매듭은 끝이 아니야. 너는 언제든 다시 풀어서 다시 시작할 수 있어.

정답이 아니야. 네 속도가 맞아.
지금이 아니어도 괜찮고, 시작했다가 풀어도 괜찮아.

[응, 매듭을 지을래]  [지금은 아니야]
```

### 사용자 반응 (시뮬레이션)
- *"끝내라고 안 시키네"* — 가역성 어휘가 결정 압박을 즉시 풀어줘.
- *"고맙다고 하네"* — D+30까지 견딘 사람에게 전해야 할 정확한 한 줄.
- v1에서 가장 약했던 *"졸업"* 어휘가 **"매듭"**으로 바뀌면서 사용자 인식이 *정거장 → 통과*가 됨. 출시 가능 수준의 카피.

## 시나리오 5 — [기록] 탭 첫 진입 (D+25)

### 보이는 카드
1. 일기 모아보기 — *오늘까지 적은 일기*
2. 추억 — *사진·메시지·장소 정리*
3. 회복의 결 — *처음과 지금의 마음 결을 비교해봐*
4. 질문통 — *미리 준비된 질문 중에서 골라 답해봐*
5. 내가 답한 질문 — *그동안 답해온 자유 질문 모아보기*
6. 매듭의 흔적 — *지나온 사이클들을 시간 순으로*

### 사용자 반응 (시뮬레이션)
- *"일기랑 답한 질문 뭐가 다르지?"* — 둘 다 시간순 텍스트 archive로 인식. 차이가 라벨에서 안 보여.
- *"질문통은 입력 화면? 답한 질문이 출력 화면? 그럼 한 화면에 있어야 하지 않나?"* — 평행 라벨이라 인풋/아웃풋 관계가 안 박힘.
- *"기록 탭이 추억을 *포함*해? 아니면 추억은 따로 있는 거야?"* — 카테고리 위계 모호.

→ 첫 사용자는 *각 카드를 한 번씩 다 눌러보고 학습*해야 해. **6개 카드의 비용**이 체감됨.

## 시나리오 6 — 페르소나 우선 카드가 떴다 (D+8)

### 사용자 반응
- *"이 카드가 왜 떴지?"* — 페르소나 라벨 비노출 정책으로 *근거*가 안 보임.

- 임상적으로 옳은 결정이지만 사용자 신뢰 측면에선 *왜* 한 줄이 필요해.

## 사용자가 "한 번 더" 만지길 바라는 것

1. **late_night CTA가 호흡 화면으로 가게.** — 한 번의 약속 이행이 다른 모든 안전 정책에 *진심* 도장을 찍어.
2. **온보딩 4단계 안으로.** — 동의·OAuth·기분·페르소나(짧게) → 나머지는 D+1~3 푸시로 분산.
3. **[기록] 탭 5카드로.** — 질문통 안에 `미답`/`답함` 탭.
4. **PersonaPriorityCard에 *왜 떴는지* 한 줄.** — 알고리즘 불투명성 해소.
5. **신체 신호 다시 접는 토글.** — *되돌릴 자유*가 작은 신뢰.
