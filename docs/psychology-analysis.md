# 심리학자 관점 분석 — Reason의 이별 회복 흐름

> **결론**: 이 앱은 "결정 보조 도구"로는 잘 짜여 있지만 "애도 회복 도구"로서는 비어 있는 부분이 많다.
> 사용자가 *무엇을 결정할지* 돕는 트랙(분석/나침반/졸업)은 5단계 이상으로 정교한데,
> *상실의 슬픔을 통과하는 과정* 자체를 다루는 트랙이 거의 없다.

---

## 1. 현재 사용 중인 심리학적 프레임 (긍정적 부분)

| 요소 | 심리학적 근거 |
|------|--------------|
| **7일 졸업 유예** | 행동심리학의 "쿨링오프(Cooling-off period)" — 충동적 종결 방지. 이별의 "회피적 종결(avoidant termination)"을 차단함 |
| **방향 선택 ≠ 비난** | 양가감정(ambivalence)의 정상화 — Bowlby 애착이론에서 "분리 항의(separation protest)"의 자연스러움 |
| **"정답이 아니야" 필수 문구** | 인지행동치료(CBT)의 *흑백사고* 차단 |
| **감정 온도 1~10** | 정서 라벨링(affect labeling) — UCLA Lieberman의 fMRI 연구상 편도체 활성을 낮춤 |
| **GPT 공감 응답 2~3문장** | Rogerian 무조건적 긍정적 존중에 가까움 |
| **"D+N" 전역 표시** | 시간 경과 인식 — "회복은 직선이 아니라 시간이 걸리는 일"이라는 메타인지 신호 |

## 2. 가장 큰 구조적 한계 — "애도(Grief)의 5단계가 없다"

Kübler-Ross의 5단계(부정→분노→타협→우울→수용) 또는 더 현대적인 Worden의 4과제(상실 인정→고통 통과→적응→재배치)에 비춰보면, 현재 앱이 다루는 감정 스펙트럼은 다음이 전부다:

```
[감정 온도 1~10] + [잡고싶어 / 보내고싶어 / 모르겠어]
```

이별 직후 사람들이 실제로 겪는 감정은 훨씬 다층적이다:

| 실제 자주 나타나는 감정 | 현재 앱 처리 |
|--------------------|------------|
| 분노/배신감 | ❌ 없음 (방향만 묻고 끝) |
| 죄책감/자책 | ❌ 없음 |
| 안도감 (관계가 힘들었던 경우) | ❌ 없음 |
| 충격/현실 부정 | ❌ 없음 |
| 외로움 vs 자유로움의 양가성 | △ "모르겠어"로 뭉뚱그림 |
| 신체 증상(수면/식욕) | ❌ 없음 |
| 자존감 손상 | ❌ 없음 |

### 개선 제안 (2-1): 일기 감정 입력 다층화

`/journal/index`의 감정 입력을 다층화:

- **1단계**: 감정 온도 (현재 유지)
- **1.5단계**: 감정 라벨 다중 선택 (분노/슬픔/죄책감/안도/외로움/그리움/혼란/허무/배신감…)
- **1.6단계**: 신체 신호 체크 (잠 못 잠/식욕 변화/멍함/울음 빈도) — 옵션

`/onboarding/mood`에서 이미 다중 감정 Pill을 쓰고 있으므로, 그 컴포넌트를 일기 작성에도 그대로 재사용하면 된다. 작은 변경으로 큰 회복 효과를 얻을 수 있는 지점.

## 3. "잡고싶어 / 보내고싶어 / 모르겠어"의 위험성

### 문제: 이 3지선다는 *집착-거리두기 차원* 하나만 측정한다

심리학적으로 이별 후 마음은 **최소 2개 차원**으로 작동한다:

```
                  보내고 싶다
                       │
   미워하면서 ─────────┼───────── 좋아하면서
                       │
                  잡고 싶다
```

현재 앱은 수직축(잡기↔보내기)만 묻고 수평축(애정↔원망)을 묻지 않는다. 그래서 이런 케이스를 구분하지 못한다:

| 상태 | 심리적 의미 |
|------|-----------|
| 보내고 싶지만 여전히 좋아함 | 건강한 수용 진입 |
| 보내고 싶고 미움도 큼 | 분노 단계 통과 중 |
| 잡고 싶지만 미워함 | 가장 위험한 상태 (상호의존/집착) |
| 잡고 싶고 좋아함 | 미해결 애착 |

이 4가지는 임상적으로 완전히 다른 처치가 필요한 상태인데, 앱은 모두 "잡고싶어" 또는 "보내고싶어"로 묶는다.

### 개선 제안 (3-1): 수평축 슬라이더 추가

`/journal/direction` 또는 `/compass/want`에 **수평축(원망 ↔ 애정) 슬라이더 1개를 추가**:

- 나침반 verdict 5종이 더 정확해짐
- 분노 단계 사용자에게 "회복 진행 신호"를 줄 수 있음
  - 예: "잡고 싶다는 마음이 줄고, 원망도 줄고 있어요"

## 4. "유예 기간 7일"의 함정

### 문제 1: Day 1~6이 *방치* 구간이 된다

유예 기간은 감정이 가장 출렁이는 구간이다. 결정을 내린 직후의 인지부조화(cognitive dissonance)가 가장 큰 시점이기도 하다. 알림을 안 주는 것은 맞지만, **앱을 열었을 때 받는 경험**은 더 풍성해야 한다.

현재 구조:
```
/cooling/index → "자율 체크인" 버튼 + 텍스트 입력 → router.back()
```

이게 7일 내내 같은 화면이다.

### 문제 2: Day별 차이가 없다

Day 1과 Day 6의 사용자 심리는 완전히 다른데, `/cooling/index`는 모든 날에 같은 화면을 보여준다. `isDay7`만 분기한다.

### 개선 제안 (4-1): Day별 회복 작업(grief work)

*알림 없이, 사용자가 앱을 열었을 때만 보여주는 방식 — CLAUDE.md 절대 규칙(Day 1~6 알림 금지) 위반 없음*

| Day | 심리적 단계 | 권장 콘텐츠 |
|-----|-----------|-----------|
| 1 | 충격/안도의 혼재 | "오늘은 아무것도 결정하지 않아도 돼" — 호흡 가이드 1개 |
| 2 | 결정 후회 가능성 ↑ | 처음 일기 회상: "그날 너는 이렇게 썼어" |
| 3 | 분노 기능 활성화 가능 | 상대방 단점/관계의 한계 회상 (이미 `pros-cons` 데이터 있음) |
| 4 | 슬픔 깊어짐 | 7일 감정 차트 + "이건 정상적인 그래프야" 메타 메시지 |
| 5 | 의미 재구성 시작 | "이 관계에서 너는 무엇을 배웠어?" 1문장 질문 |
| 6 | 미래 상상 | "졸업 후 첫 한 주를 어떻게 보내고 싶어?" |
| 7 | 통합/확정 | 현재 화면 유지 |

### 문제 3: 체크인이 *일방향 입력*이다

`/cooling/checkin.tsx`는 텍스트 입력 → 저장 → back. 사용자가 쓴 글에 어떤 응답도 오지 않는다.
일기 작성 트랙(`/journal/response`)에서는 GPT 공감 응답을 주는데, 가장 취약한 시점인 유예 체크인에서는 응답을 안 주는 건 비대칭이다.

### 개선 제안 (4-2): 체크인 GPT 응답 추가

`/cooling/checkin`에도 GPT 공감 응답을 추가하되, 톤을 "결정을 흔들지 않는" 방향으로 잡는다:
> "지금 그 마음, 유예 기간이 알려주는 정상적인 신호야. 7일째에 다시 물을게."

## 5. "분석/나침반"이 너무 *인지적*이고 너무 *조기에* 등장한다

### 진입 시점 문제

홈에서 바로 `/(tabs)/analysis`, `/(tabs)/compass`로 갈 수 있다. 이별 다음 날 진입한 사용자가 곧바로 "장단점 입력 → 슬라이더 → 진단 결과"를 받는다.

이별 직후 2~3주는 인지 처리 능력 자체가 떨어진 상태다 (스트레스로 인한 working memory 저하). 이때 받는 "재결합 가망 35%, 회복 가망 67%" 같은 수치화된 진단은:

- 통제감(sense of control)을 줄 수도 있고
- **반추(rumination)를 가속**할 수도 있다 — 더 위험한 쪽

### 문제: 진단 결과가 *고정*된다

`upsertRelationshipProfile`로 저장하면 "내 회복 가망은 67%야"가 사용자 머릿속에 고정 핀이 박힌다. 이 수치는 D+3과 D+30에 완전히 달라야 하는 수치다.

### 개선 제안 (5-1): 시간 게이트

```
D+0 ~ D+7   → 분석/나침반 진입 시 부드러운 만류
              "지금 결정하지 않아도 돼. 일주일은 그냥 흘려보내자."
              (강제 차단은 X — 자율성 존중)

D+8 이상    → 정상 진입
```

이미 `canGraduate(D+30 & 일기 5개)` 같은 게이트가 졸업에 있다. 같은 패턴을 분석/나침반에 적용.

### 개선 제안 (5-2): 진단 결과의 *시간성* 명시

`/analysis/result`에서 결과를 보여줄 때:

> "이 수치는 D+12 시점의 너야. 한 달 뒤엔 다른 결과가 나올 거야."

이 한 줄이 고정 마인드셋을 막는다. CLAUDE.md의 "정답이 아니야" 규칙과 정확히 같은 결의 보강이다.

## 6. "관계 분석"의 장단점 입력이 *대칭적*이다 — 심리학적 함정

### 문제: 장점/단점 동등 가중

`/analysis/pros-cons`는 장점 탭과 단점 탭으로 나뉘어 동등하게 입력받는다.

이별 직후 인지편향은 시간에 따라 다르게 작동한다:

```
D+1 ~ D+14:  "장미빛 회상(rosy retrospection)" — 장점만 떠오름
D+15 ~ D+30: 양가감정 — 두 측면 모두 보임
D+30 이후:   안정화 — 단점도 객관화됨
```

현재 앱은 이 시간차를 무시하고 동시에 입력받는다. 그래서 이별 직후 사용자는 장점만 잔뜩 적고, 그게 누적되어 분석 결과를 왜곡시킨다.

### 개선 제안 (6-1): 시점별 장단점 분리 누적

장점/단점을 시점별로 분리해 누적시킨다:
- "오늘 떠오른 장점 1개"를 일기에 흡수
- 2주 뒤 그것들을 모아서 보여주는 방식

`relationship_profile`이 이미 누적 구조라 데이터 모델 변경 없이 가능하다.

## 7. 가장 큰 누락 — *애도 의식(grief ritual)*과 *추억의 처리*

### 문제: 추억 처리 트랙이 0개다

현재 앱에는 상대방과의 추억을 다루는 트랙이 없다. 사진, 메시지, 함께 갔던 장소 등 **떠올리는 트리거**에 대한 처리가 없다.

이별 회복에서 가장 자주 호소되는 게 "갑자기 떠오를 때를 어떻게 처리할지 모르겠다"인데, 앱은 사용자가 자발적으로 일기 화면에 들어와야만 처리할 수 있는 구조다.

### 개선 제안 (7-1): *떠오름(intrusive memory) 처리* 빠른 진입점

홈에 작은 버튼 1개 추가:
> 🫧 "지금 갑자기 떠올랐어"

탭하면 30초~1분짜리 짧은 진정 흐름:
1. 호흡 1회 가이드
2. "그 기억은 사실이지만, 지금의 너는 그 기억 속이 아니야"
3. 감정 온도 입력 → 끝

이는 **DBT(변증법적 행동치료)의 distress tolerance 모듈**과 정확히 같은 구조다. 코드량도 작고 효과는 크다.

### 개선 제안 (7-2): 졸업 의식의 능동성 확보

현재 `/graduation/letter`는 AI가 써준 편지를 *읽기만* 한다. 심리학적으로 물리적/상징적 마무리 행위(편지 쓰기, 사진 정리, 한 곳에 모아 닫기)가 회복에 결정적이다. AI가 써준 편지를 읽기만 하는 구조는 사용자가 능동적으로 의미 부여를 할 기회를 뺏는다.

`/graduation/letter`를 **사용자가 직접 쓰는 한 줄짜리 작별 문장 + AI가 그것을 받아 마무리**하는 양방향 구조로 전환해야 한다.

## 8. 기술적 구현이 심리학적 의도를 배반하는 지점들

### 8-1. 임시 저장 없음

> *일기 작성 중 탭 이동 시 입력 손실 가능* (`user-flow.md` K-2)

이별 일기를 한참 쓰다 알림 하나에 잃으면 재시도 의지가 꺾인다. 회복 도구에서 가장 비싼 자산인 *재방문 의지*를 손상시킨다. AsyncStorage로 draft 저장만 추가해도 충분하다.

### 8-2. 에러 복구 경로 불명확

> *분석/졸업 저장 실패 시 재시도 UI 없음 (console.warn만)* (`user-flow.md` K-4)

졸업 신청을 눌렀는데 조용히 실패하는 건, 사용자 입장에서 "내 결단이 받아들여지지 않았다"는 미묘한 거부 신호로 작동할 수 있다. 졸업 같은 의식적 행위는 **확정적 피드백**이 필수다.

### 8-3. 일기 4단계 흐름이 너무 길다

```
mood → direction → question → response (4단계 + AI 스트리밍)
```

이별 직후 무기력 상태(D+1~7)의 사용자에게 매일 4단계는 진입 장벽이다. "오늘은 감정 온도만 찍고 나가도 되는" 짧은 모드(`mini journal`)가 있어야 한다. 일기 5개를 채워야 졸업이 가능한 게이트(`canGraduate`)와 연결하면 더 효과적이다.

*같은 노력 대비 회복 효과가 큰 순서*

| 순위 | 개선 항목 | 심리학적 근거 | 작업 규모 |
|-----|----------|-------------|---------|
| **1** | 일기 감정 입력 다층화 (분노/죄책/안도 추가) | 애도 모델 정상화, 정서 라벨링 효과 | 작음 — mood Pill 재사용 |

| **5** | 일기 임시 저장 + 미니 모드 (감정 온도만) | 재방문 의지 보호, 진입 장벽 낮춤 | 작음 |

## 10. 철학적 보완 — "감정의 안전장치" 부재

CLAUDE.md 절대 규칙들은 **결정의 안전장치**(graduation, 7일 유예, 비난 금지)에 집중되어 있다. 이건 훌륭하다.

그런데 **"감정의 안전장치"**가 빠져 있다:

| 상황 | 현재 처치 | 필요한 처치 |
|------|---------|-----------|
| 감정 온도가 3일 연속 1~2점 | ❌ 없음 | 부드러운 안녕 확인 + 자원 안내 |
| 한밤중(00:00~04:00) 진입 | ❌ 없음 | 이별 후 새벽 진입은 임상적 위험 신호 |
| "혼자가 아니라는 신호" | ❌ 없음 | 익명 통계 ("이 시점에 비슷한 마음을 적은 사람이 N명 있어") |
| 위기 자원 | ❌ 없음 | 전문 상담 핫라인 옵트인 안내 |

이는 절대 규칙급으로 다뤄야 한다. 자살 위험 스크리닝이나 위기 자원 안내(전문 상담 핫라인)는 *옵트인 형태*로라도 있어야 한다.
