# Quality And Risks

## 현재 핵심 리스크
- 진단 점수 매핑/경계값 명세 미흡 시 결과 일관성 저하
- 질문 재노출 규칙 부재 시 UX 피로도 증가
- enum 혼용 시 데이터 해석 오류 위험
- 유예 리셋/취소 데이터 정책 미정 시 리포트 왜곡 위험

## 필수 보완
- 계산식/경계값 명세 고정
- 질문 추천 스코어링 규칙 고정
- 스키마 unique + upsert 강제
- 실패 fallback과 지연 UX 확정

## 운영 지표
- 일기 7일 유지율
- 유예 완주율/취소율/복귀율
- 졸업 확정률
- 알림 품질(클릭률/무응답률/차단율)

## QA 최소 세트
- 방향 급변 케이스
- 3일 연속 동일 방향 케이스
- 유예 리셋/취소/재신청 케이스
- GPT 실패/타임아웃 fallback 케이스
