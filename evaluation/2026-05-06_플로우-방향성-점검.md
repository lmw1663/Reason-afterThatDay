## TL;DR — 한 문단으로

**방향성의 큰 줄기(페르소나별 1차 장애물 분리 + 단계적 게이트 + 7일 유예 + C-SSRS 우선)는 맞다.** 다만 *제품 약속*("단순한 결정 지원 도구")과 *실제 구현*(20페르소나 × 200셀 클리니컬 분기 엔진) 사이에 점점 벌어지는 간극, 그리고 절대 규칙("단정 금지") vs 화면 표시("재연결 가능성 35%") 사이의 직접적 모순이 4건 존재한다. 출시 전 *반드시* 손봐야 할 블로커가 4개, 방향 자체를 흔들 수 있는 충돌이 3개, 톤 조정 수준의 폴리싱이 다수다.

## 1. 방향성이 *맞다*고 평가되는 부분

### 1-1. 페르소나 분기의 심리학적 골격 ✅
| 평가 | 출처 |
|------|------|
| "1차 장애물 × 안전" 분리 원칙 자체는 임상적으로 정당함 | `페르소나-분류체계.md:250` "C-SSRS 양성 > 페르소나 분기 > 표준" |

### 1-2. 절대 규칙이 *문서적*으로는 잘 박혀 있다 ✅

### 1-3. 시간 게이트의 분리가 합리적이다 ✅
- P03 추억 D+21 / P14 자기 용서 D+60 / P02 나침반 D+10 / P04 D+14 / P07 D+21
- "충동기 회상 차단" / "수치심기 자기 용서 차단" / "충격기 결정 차단"의 *이유*가 명확

### 1-4. 다중 페르소나 처리의 *Guard Overlay* 개념 ✅

## 2. 방향성이 *흔들리는* 부분 (핵심 우려)

### 2-1. 🚨 "단순 결정 지원" vs "20페르소나 임상 분류 엔진"의 약속 충돌
**심각도: HIGH — 방향 자체를 결정해야 함**

| 약속 | 현실 |
|------|------|
| `CLAUDE.md`: "결정 지원 도구, 진단 아님" | 8축 점수 + top-2 페르소나 + 200+셀 분기 + 재분류 cron |
| `01-product-principles.md`: "짧고 따뜻한 문장" | P14는 "수치심 ≠ 죄책감" 심리교육 카드, P19는 ROCD ERP 차단, P20은 도파민-코르티솔 사이클 안내 |
| 라벨 비노출 (코드 안 보임) | *경험*은 페르소나마다 80% 다름 — 사용자는 *왜* 다른지 모르는 채 다른 앱을 쓴다 |

**3개 평가자(임상·UX·사용자관점)가 공통 지적**:
- 사용자가 P01~P20을 보지는 못해도, **시스템은 진단을 하고 있다**
- "이 페르소나 분기는 필요한가?"라는 질문이 `TODO.md`에서 자체적으로 제기됨
- 첫 사용자에게는 *복잡한 임상 평가 엔진*이 *간단한 위로 도구*로 가장한 것처럼 보임

**판단**: 이건 *실행 디테일*이 아니라 **방향 결정**이다.
- (a) 임상 깊이로 commit → 임상 검증·치료 대체 고지·치료자 권유 트랙 필수
- (b) 단순 결정 지원으로 회귀 → 페르소나 분기를 *3~4개 큰 유형*으로 통합 (A 안전 / B 회피 / C 충동 / D 의미)

현재 페르소나별-유저-플로우.md는 *방향(a)을 implicit하게* 선택한 상태. 그러나 외부 약속(`CLAUDE.md`, 마케팅 톤)은 *방향(b)*에 가깝다. 둘 중 하나로 모아야 한다.

### 2-2. 🚨 "단정 금지" 절대 규칙 vs "재연결 가능성 35%" 화면 표시
**심각도: BLOCKER — 출시 전 수정 필수**

- `CLAUDE.md:14`: "진단/나침반 결과 화면에 '정답이 아니야' 문구 필수"
- `01-product-principles.md`: "단정 금지 / 가능성 제시 / 비난 금지"
- 그런데 `analysis/result.tsx`는 **"재연결 가능성 35%"** 같은 정량화 메터를 표시
- **3명 평가자 모두 동일 지적**: "정답이 아니야"는 작은 회색 텍스트, "35%"는 큰 차트 → 시각 위계로 disclaimer가 무력화됨

**페르소나별 플로우 문서엔 이 모순이 빠져 있다.** P01·P14·P20에 분석 트랙 차단을 거는데, *나머지 페르소나*에게는 이 정량화 메터가 그대로 노출된다. 분석 차단이 일부 페르소나에게는 *오히려 더 안전한 출구*가 되는 역설.

**필수 수정 방향**:
- 정량 % 라벨 완전 제거 OR 추세선(이동평균)으로 변환
- 7일 방향 변화 횟수만 *사실*로 표시, 가망 점수는 삭제

### 2-3. 🚨 매일 일기 → 반추(rumination) 강화 위험
**심각도: HIGH — 페르소나별 분기와 충돌**

- 졸업 게이트가 *일기 5개*를 요구함 → "매일 쓰는 게 좋아" 톤으로 자연스럽게 흐름
- Nolen-Hoeksema 반응 양식 이론: **반복적 자기초점 = 우울 위험 증가**
- 특히 **P03(불안형) / P10(분노) / P14(외도 가해)**처럼 반추 취약 페르소나에게 매일 쓰기 압박은 *iatrogenic*(돕는 척하며 해를 끼침)

**현재 페르소나별 플로우의 처리**:
- P02만 미니 모드 default 설정 (가벼운 진입)
- 그 외는 모두 풀 일기 default → 반추 위험 페르소나에 별도 가드 없음
- 7일 연속 작성 감지 → "오늘은 쉬어볼래?" 같은 *역방향 넛지*가 없음

**필수 수정 방향**:
- 페르소나별 *권장 빈도*를 `personaBranches.ts`에 추가 (P03·P10·P14는 격일 권장, P02는 미니, 등)
- 7일 연속 작성 시 푸시: "매일 쓰고 있네. 하루 쉬는 것도 회복이야"
- 졸업 게이트의 "일기 5개"는 그대로 두되, *연속*이 아닌 *총 횟수*로 (이미 그런 듯하지만 톤이 "매일"로 들림)

### 2-4. 🚨 "졸업" 메타포 vs 현대 grief 이론
**심각도: MEDIUM — 사용자 기대치 붕괴 지점**

- "졸업" = 완료, 끝, 통과
- Bonanno 회복 곡선·Continuing Bonds 이론: **grief는 평생 통합 작업**, 완결되지 않음
- 사용자가 D+30에 "졸업"하고 친구에게 "이별 정리됐어"라고 말한 뒤, 3주 후 grief wave가 오면 "졸업했는데도 슬픈 나는 실패한 건가?" → 자책 trigger

**페르소나별 플로우 문서의 누락**:
- 졸업 후 wave에 대한 안내 (D+30·D+90 "사실은 평생 weight를 지는 거야" 메시지) 없음
- P17(강제 이별)에는 Continuing Bonds 트랙이 있는데, *모든 사용자에게* 이 개념을 확장하는 게 임상적으로 더 정확

**수정 방향**:
- "졸업" 라벨 자체를 *유지*하되(이미 절대 규칙 + 익숙한 메타포), 졸업 후 D+30·D+90에 "재방문 회상" 푸시 (P05만 있는 거 → 모든 사용자로 확장)
- 졸업 화면 카피에 "끝이 아니야. 첫 발걸음이야." 명시

### 2-5. ⚠️ 페르소나 분기의 *비대칭*: 분석/나침반 차단 페르소나의 출구
**심각도: MEDIUM**

- P01·P14·P20: 분석 트랙 *전체* 차단 + 나침반 footer 강제 ("너의 잘못이 아니야")
- P17: 나침반 영구 비활성

→ **그럼 이 사용자들이 졸업까지 가는 길은?**

페르소나별 플로우 문서에는 이들의 about-me 트랙·일기·추억까지는 적혀 있지만, **"졸업 신청까지의 경로"**가 없다. 분석/나침반이 차단된 사용자가 7일 유예 화면에 어떻게 도달하는지 명시되지 않음. 코드 가드는 있지만 *플로우 다이어그램*이 빠진다.

**수정 방향**:
- P01·P14·P17·P20 전용 "졸업 경로" 섹션 추가 (about-me 카테고리 ≥ N개 작성 + 일기 5개 → 졸업 권장 카드)

## 4. 페르소나별 유저 플로우 문서 자체의 보완점

방금 만든 `docs/psychology-logic/페르소나별-유저-플로우.md`는 *현재 코드*는 정확히 반영하지만, 위 분석을 거쳐 보면 다음을 추가하는 게 좋다:

### 4-1. 누락된 항목
- [ ] **각 페르소나의 졸업 경로** — 분석/나침반 차단 페르소나(P01·P14·P17·P20)는 어떻게 졸업까지 가나?
- [ ] **각 페르소나의 위기 핫라인 매핑** — 매트릭스 C10에 있는데 코드에는 미반영, 문서엔 누락
- [ ] **권장 일기 빈도** — 현재 모든 페르소나가 "매일" 톤. 반추 취약 페르소나(P03·P10·P14)에 격일 권장 명시 필요
- [ ] **졸업 후 wave 안내** — D+30·D+90 재방문 메시지를 P05 외 모든 페르소나로 확장 필요

### 4-2. 현재 문서의 *암묵적 합의*가 외부 평가와 충돌하는 점
| 페르소나별 플로우 문서의 가정 | 외부 평가의 우려 |
|--------------------------|---------------|
| 페르소나 분기는 정밀한 게 좋다 (200+셀) | 사용자에게 보이는 건 결국 1~2개 카드. 정밀도가 *복잡한 코드 유지비* + *unverified 임상 가정*으로 이어짐 |
| 라벨 비노출이면 진단 아님 | 시스템이 진단하는데 사용자가 모르는 게 *오히려* 윤리 문제일 수 있음 |
| GPT 톤·금기 키워드는 lint로 충분 | OpenAI 전송 자체가 *명시 동의 없이* 진행되는 게 더 큰 이슈 |

## 5. 결정해야 하는 갈림길 (방향 결정)

### 갈림길 A: 임상 깊이로 commit
- 페르소나 20개 유지
- 트라우마 스크리너 + 치료 대체 고지 + 치료자 권유 트랙 추가
- 임상 자문 + 사용자 검증 (IRB-style 또는 적어도 1차 사용자 인터뷰 N=20)
- 마케팅 톤을 "결정 지원" → "회복 동반자"로 조정
- **장점**: 차별화. 임상적 정당성. **단점**: 검증 비용·시간·법적 리스크.

### 갈림길 B: 단순 결정 지원으로 회귀
- 페르소나를 4유형(A/B/C/D)으로 통합
- 매트릭스 200셀 → 4×N(셀) 단순화
- "치료가 아닌 결정 지원" 마케팅 톤 강화
- 클리니컬 메시지(수치심 ≠ 죄책감, 도파민 사이클 등) 약화 또는 외부 자원으로 위임
- **장점**: 단순·검증 부담 적음·약속과 정합. **단점**: 차별화 약화.

### 갈림길 C: 하이브리드 (추천 가능)
- 표면 UX는 단순 (4유형 카드만 노출)
- 내부 분류는 20페르소나 유지하되, **차단·보호 게이트만** 가동 (블록 리스트 우선)
- *boost·content 분기*는 페르소나 라벨 비노출 + 사용자 동의 후에만
- 임상 메시지는 *옵션 콘텐츠*로 (사용자가 "더 알고 싶어" 누를 때만)
- **장점**: 약속 유지 + 임상 안전망. **단점**: 코드 복잡도 그대로.

**현재 코드는 사실상 갈림길 C에 가까운 상태인데, 문서·약속이 그걸 명시하지 않아서 일관성이 깨져 보인다.** 의도가 C라면 그걸 `CLAUDE.md`나 `docs/guide/01-product-principles.md`에 명시적으로 박는 게 좋다.

## 6. 우선순위 정리 (실행 순서)

### Phase 1 (출시 전 BLOCKER) — 1~2주
1. 정량화 회복 % 라벨 제거 또는 추세선화 (`analysis/result.tsx`)

6. 새벽 모달 CTA 정합성 (호흡 화면 신설 또는 3옵션 모달)

### Phase 2 (출시 후 1개월) — 방향 정렬
1. 갈림길 A/B/C 중 명시적 선택 → `CLAUDE.md` 갱신

## 7. 한 줄 결론

> **방향성의 큰 줄기는 맞다. 다만 (a) 절대 규칙 직접 위반 4건은 출시 블로커이고, (b) "단순 vs 임상" 약속-구현 간극을 *명시적으로* 결정하지 않으면 페르소나 분기는 더 정교해질수록 더 흔들린다.** 페르소나별-유저-플로우.md 문서 자체는 *현재 코드의 거울*로서 정확하지만, 그 거울이 비추는 방향이 외부 약속과 어디서 어긋나는지 보이게 해줬다 — 그게 이 점검의 핵심 결과.
