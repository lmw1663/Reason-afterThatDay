# 심리검사 도입 가능성 분석 — Reason 앱

> **목적**: `심리검사.md`에 정리된 표준화된 임상 척도를 Reason 앱에 *어느 시점에 어떻게* 도입할 수 있는지 검토.
> **전제**: CLAUDE.md 절대 규칙(채팅 UI 금지·"정답이 아니야" 원칙·위기 모달·D+N 표시) 준수 + `페르소나.md`의 10개 페르소나 분기와 정합성 유지.

## 2. 시점별 검사 활용 시나리오

### ① 온보딩 (D+0) — *초기 스크리닝*
- **PHQ-9 단축형(2항: PHQ-2)** → 우울 수준 빠른 게이트
- **GAD-7 단축형(2항: GAD-2)** → 불안 수준
- **C-SSRS 1~2항** → 자살사고 여부(필수, 양성이면 즉시 핫라인)
- **ECR-R 단축형(애착불안 3 + 회피 3)** → 페르소나 P2/P3 분기

**UX**: `페르소나.md`의 Q1~Q7 흐름에 *섞어* 넣음. "검사" 티 안 나게 — 한 화면 한 문장.
- 예: "지난 2주, 잠들기 어려운 날이 며칠 있었어?" (PHQ-9 #3을 화면 전환형으로)

### ② 쿨링오프 진입 (졸업 결정 시) — *위기 평가*
- **C-SSRS 전체(6항)** → 충동 결정의 위험성 차단
- 양성 시 졸업 유예를 7일 → **14일로 자동 연장**

### ③ D+7 / D+14 / D+30 — *회복 추적*
- **PHQ-9 전체(9항)** 격주 측정 → 우울 점수 곡선 시각화
- **Rosenberg 전체(10항)** D+0, D+30, D+60 측정 → 자존감 회복 추적
- **GAD-7** D+7, D+30 측정

**UX**: "오늘은 너의 회복 상태를 같이 확인해보자" 한 화면 도입 → 9문항 슬라이드 전환

### ④ 위기 모달 발동 시 (3일 연속 저온 / 새벽 접근)
- **C-SSRS 1~3항 즉시 묻기** → 강도에 따라 핫라인 자동 노출
- 현재 `EmotionalCheckModal`에 표준화된 척도가 없음 → C-SSRS로 객관적 트리거 도입

### ⑤ D+30 이후 — *반추 패턴 점검*
- **RRS 단축형(10항)** → 곱씹기 패턴 식별
- 점수 높으면 P3(불안형) / P5(죄책감형) 가중치 ↑
- 인지 재구조화 워크시트 자동 추천

### ⑥ 졸업 시점 — *종합 평가*
- PHQ-9 + GAD-7 + Rosenberg 동시 측정 → "출발(D+0) vs 졸업" 비교 카드
- 회복 곡선 시각화 → 사용자에게 *진전의 증거* 제공

### ⑦ 졸업 후 D+60, D+90 — *복합비애 확인*
- **ICG (복합비애)** → 6개월 이상 지속되는 비정상 비애 식별
- 양성 시 *전문가 상담* 권유 (앱 한계 명시)

## 3. 인터랙티브 변환 원칙 (CLAUDE.md 준수)

`심리검사.md`에서 "딱딱하지 않게 인터랙티브하게"라고 명시한 부분 — 다음 4가지 변환 패턴 제안:

1. **1문항 = 1화면 (화면 전환형)** — 채팅 UI 금지 규칙 자동 충족
2. **임상 표현 → 일상어 번역**
   - PHQ-9 #1: "Little interest or pleasure in doing things"
   - → "요즘 평소에 즐겁던 일이 시들해진 날이 있어?"
3. **5점 리커트 → 감정 슬라이더** (이미 앱에 있는 `MoodSlider` 재사용)
4. **검사 결과 = 점수 X / 색·이미지 O**
   - 점수 노출 금지(자기진단 위험)
   - "지금 너의 마음 날씨는 흐림" 정도로 메타포 처리

## 4. 페르소나 ↔ 검사 매핑

| 페르소나 | 핵심 검사 | 사용 목적 |
|---------|---------|----------|
| P1 가스라이팅 | Rosenberg + RRS | 자존감 손상·자기 검열 추적 |
| P2 회피형 | ECR-R 회피 차원 | 회피 강도 확인 → 일기 미니 모드 자동화 |
| P3 불안형 | ECR-R 불안 차원 + RRS | 매달림·반추 추적 |
| P4 갑작스러운 통보 | PHQ-9 + Rosenberg | 충격 후 우울·자존감 손상 |
| P5 죄책감형 | RRS + PHQ-9 | 자책 반추 추적 |
| P6 반복 사이클 | ECR-R + RRS | 패턴 식별 |
| P7 첫 이별 | PHQ-9 + GAD-7 | 강도 모니터링 (회복 가능성 시각화 근거) |
| P8 장기 권태 | PHQ-9 + Rosenberg | 정체성 손상 추적 |
| P9 헌신 소진 | Rosenberg + ECR-R | 자아 회복 측정 |
| P10 분노 지배 | GAD-7 + C-SSRS | 각성·충동성 모니터링 |

## 5. 권장 도입 순서

### Phase A — MVP (무료 + 즉시 가치)
1. **C-SSRS 핵심 2항** → 위기 모달에 직결 (가장 시급)
2. **PHQ-9 + GAD-7** → 온보딩 + 격주 추적
3. **Rosenberg** → 졸업 전후 비교 카드

### Phase B — 페르소나 정확도 강화
4. **ECR-R 단축형 12항** → 온보딩에 자연스럽게 흡수
5. **RRS 단축형** → D+30 이후 반추 트래킹

### Phase C — 장기 회복
6. **ICG** → D+60 이후 복합비애 스크리닝

## 7. 주의사항 (도입 전 반드시 확인)

1. **자기진단 위험** — 점수·진단명을 사용자에게 직접 노출 금지. "PHQ-9 점수 18점 = 중등도 우울"을 그대로 보여주면 자기충족적 예언이 됨.
2. **라이선스 재확인** — ECR-R·RRS는 *연구 목적* 무료. 상업 앱 사용 전 원저자/라이선스 보유 기관에 *서면 확인* 필수.
3. **임상 진단 도구 ≠ 진단** — 모든 화면에 "이건 진단이 아니야, 참고용이야" 문구 필수 (CLAUDE.md "정답이 아니야" 원칙과 일관).
4. **데이터 민감도** — PHQ-9·C-SSRS 결과는 의료정보. RLS 수준을 일반 데이터보다 한 단계 강화(별도 테이블 + 추가 암호화 검토).
5. **검사 피로도** — 같은 검사 반복 측정 시 응답 자동화 위험. 격주 이상 간격 + 매번 다른 *프레이밍*으로 반복 인식 줄이기.
6. **위기 양성 처리 프로토콜** — C-SSRS 양성 응답 시 자동 분기 명세 필수: 즉시 핫라인 노출 + 졸업 유예 자동 연장 + 일기 잠금 등.
7. **검사 거부권 보장** — 모든 검사는 *언제든 건너뛰기* 가능해야 함. 강제 흐름은 회피형(P2)·통보 피해형(P4)에게 재트라우마.
